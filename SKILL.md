# 図解化スキル（MAKASERU版）

どんなテーマでも、前提知識ゼロの人が読んで腹落ちする図解HTMLを1枚作る。品質基準は「初めて聞く人が、読み終わったら自分の言葉で説明できる明快さ」。この基準自体は出力に表示しない。

## 2つの使い方

- **A. 解説図解**：わからない言葉・仕組みを噛み砕く（例：「ディープリサーチとは何か図解して」）
- **B. 業務の見える化**：自分の仕事の流れ・工程・段取り・役割分担を1枚にする（例：「うちの受注から納品までの流れを図解して」）

Bの場合、材料が足りなければ**質問は最大2つ**まで（例：「誰に見せる図解ですか？」「流れはどこから始まってどこで終わりますか？」）。聞きすぎない。ユーザーが話した事実だけを使い、**業務内容を勝手に創作しない**。不明な部分は図解内で「（要確認）」の枠にする。

## 依存ファイル

- `references/base.html` — 図解テンプレート（Tailwind CSS CDN・Lucide Icons CDN・MAKASERU配色の「額縁」）
- `references/model-answer.html` — 模範回答（品質・デザインパターンの実例）

## ワークフロー

### Step 0: 前提確認

このスキルフォルダ内の `references/base.html` の存在を確認する。無ければ「テンプレートが見つかりません。フォルダ構成が壊れている可能性があるので、講師（MAKASERU川口）に連絡してください」と伝えて終了。

### Step 1: 模範回答を読む

`references/model-answer.html` を読み、完成品の品質水準・視覚表現（カード／ステップフロー／対比）・Tailwindクラスの語彙・情報量を把握する。ルール集ではなく実物から読み取る。

### Step 2: テンプレートを読む

`references/base.html` を読み、額縁を把握する：`<!-- TITLE -->`／`<!-- DESCRIPTION -->`／`<!-- CONTENT_START -->`〜`<!-- CONTENT_END -->` の位置、MAKASERU配色（`mk-*`）、読み込み済みCDN。

### Step 3: 材料を集める

- **A（解説図解）**: ウェブ検索を最大2〜3回。①正確な定義（公式情報優先）②最新動向③初心者に伝わるたとえに使える具体例。採用した情報の出典URLを控える
- **B（業務の見える化）**: 検索はしない。ユーザーの発言・共有された資料だけが材料。曖昧な箇所は上記のとおり最大2問で確認

### Step 4: コンテンツ生成

模範回答のデザインを参考に、Tailwindの語彙で自由に組む。構成の型：

1. **ヒーロー**：カテゴリバッジ＋タイトル＋1文の説明
2. **一枚絵サマリー**：「ひとことで言うと」の太字1文 → それを図にしたコア図解 → 身近な接点ピル2〜4個
3. **各論**：内容に応じてパターンを選ぶ
   - 定義（〜とは） → アナロジー図（身近なたとえを矢印で関係づけ）
   - プロセス・工程 → 番号つきステップフロー（各ステップにアイコン＋一言。**5±2個に圧縮**）
   - 違い・比較・Before/After → 左右対比（同じ観点を同じ行に。✗/✓と赤/緑）
   - 具体例 → カードグリッド（アイコン＋タイトル＋説明）
   - 役割分担 → 担当ごとのレーンカード（誰が・何を・いつ）
   - 数字 → 数字カード（大きく・色を変えて）
   - 時系列 → タイムライン
4. **まとめ**：持ち帰り3点＋（Aのみ）インライン出典

### Step 5: 保存して開く

1. デスクトップに `図解` フォルダがなければ作る（Windows: `%USERPROFILE%\Desktop\図解` ／ macOS: `~/Desktop/図解`）
2. テーマの短い英単語スラッグを決める（例: `juchu-flow`, `deep-research`）
3. `references/base.html` を `図解/{スラッグ}.html` にコピーし、プレースホルダー3箇所を置換する
4. ブラウザで開く（Windows: `start "" "パス"` ／ macOS: `open "パス"`）。開けない環境なら「デスクトップの『図解』フォルダの {スラッグ}.html をダブルクリックしてください」と案内する

### Step 6: 完了報告

```
完成：【図解のタイトル】

（内容の要約1〜2文）

保存先：デスクトップの「図解」フォルダ → {スラッグ}.html
（ダブルクリックでいつでも開けます。印刷やPDFにしたい時は、開いた画面の右上の「PDF」を押してください）

主なポイント：
-（3〜5個）
```

## 禁止事項

- **ReactやJSフレームワークを使わない** — 静的HTMLで完結させる
- **絵文字を使わない** — OS依存で崩れる。アイコンはLucide Icons
- **`<script>`・`<style>`を追加しない** — テンプレートに含まれるもの以外禁止。スタイルはTailwindクラスのみ
- **外部リソースを追加しない** — テンプレート同梱CDN以外の画像URL・フォント・CDN禁止
- **額縁構造を変えない** — `<head>`・CDN・metaはそのまま
- **アニメーション・開閉UI・フォームを入れない** — 印刷とPDFで壊れる
- **読者のレベルをラベリングしない** — 「初心者向け」等をタイトル・見出しに書かない。噛み砕くのは品質であって読者のレベルではない
- **`bg-clip-text text-transparent` のグラデ文字に依存しない** — PDFで消える（printフォールバックはあるが単色になる）

## 生成の指針

- 概論 → 各論。いきなり詳細に入らない
- 専門用語は初出で括弧書き解説（例：「トークン（AIが文章を数える時の単位）」）
- たとえ話は町工場・料理・日常の道具など、体で知っているものに寄せる
- 図を先に見せて、説明は図の後ろに置く。見出しから最初の図までテキストは最大2段落
- 簡潔にしすぎない。腹落ちに必要な情報量は削らない
- 全部日本語で書く
