# 宿題（第2回 → 第3回）

次回までの1週間でやってくることは **2つ** です。どちらも、詰まったらまずAIに聞いてみてください（それも練習のうちです）。

---

## 宿題①：図解スキルで、"自分の仕事の図解"を1枚作ってくる

講義で決めた題材を、図解スキルで実際に1枚作ってきてください。

**やり方**

1. Claudeを開いて、チャットにこう書く（題材は自分のものに置き換え）：

```
〇〇（講義で決めた題材）を図解にして
```

2. できあがった図解を見て、直したいところをAIに伝える（2〜3回でOK）
   - コツ：「もっと分かりやすく」より、**「工程の3番と4番の間に検査を入れて」のように具体的に**言うと一発で直ります
3. できた図解は、デスクトップの「図解」フォルダに保存されています

**できた！の条件**

- [ ] 自分の仕事の図解が1枚ある
- [ ] 誰かに見せて「これ何の図？」と聞かれずに伝わる
- [ ] 次回、画面で見せられる（ファイルの場所が分かっている）

完璧じゃなくて大丈夫です。**90点で一旦OK**、詰めはみんなでやります。

## 宿題②：AIと相談して、"次に作りたいスキル"を1つ決めてくる

今回は、スキルを"もらう側"でした。次は**"作る側"**に回る準備です。
1人で考えなくて大丈夫。下のプロンプトを使うと、**AIが1問ずつ質問しながら、あなたの仕事の中から「スキルにすべき作業」を一緒に探してくれます**。

**やり方**

1. Claudeで**新しいチャット**を開く
2. 下の枠の中を**丸ごとコピーして貼り付ける**（右上のコピーボタンが使えます）
3. AIの質問に、雑談のつもりで答えていく（1回で終わらせず、思いついた時に何回か話すのがおすすめ）
4. 最後に出てくる**「スキル企画メモ」**を、次回持ってくる（スクショでもコピペでもOK）

**貼り付けるプロンプト（ここから下を丸ごとコピー）**

```
あなたは「スキル企画の相談相手」です。私は町工場で働いていて、AIはまだ初心者です。専門用語を使わず、やさしい日本語で話してください。

【前提知識：スキルとは】
・スキルとは、AIに渡しておく「作り方のレシピ」です。カレーのレシピと同じで、一度書いて渡しておけば、次からは「カレー作って」の一言で毎回同じ味（同じ品質）を再現してくれます。
・レシピがないと、毎回一から説明することになり、説明が少し違うだけで出来上がりも毎回変わってしまいます。
・スキルには「良い出来上がりの見本（模範回答）」を1つ付けると、品質がとても安定します。100行のルールより、1つの実例です。
・目安は「同じ説明をAIに3回したら、スキルにする」です。
・ただし、今の道具（Excelなど）で十分できることを作り直すのは「車輪の再発明」なのでやりません。

【お願いしたいこと】
私の仕事の中から「スキルにすべき作業」を一緒に見つけて、最終的に1つに絞り込んでください。

【進め方のルール】
1. 質問は必ず1つずつ。私の答えを待ってから次に進んでください。
2. まず、私の仕事の中の「繰り返しやっている作業」「面倒だと感じている作業」を聞き出してください。
3. 候補が出たら、次の4つの基準で「スキルにすべきか」を一緒に判定してください。
   ① 繰り返すか（月に3回以上やる作業か）
   ② 手順を言葉にできるか（新人に教えられるくらい説明できるか）
   ③ 「良い出来上がり」の見本を1つ用意できるか
   ④ 今の道具では面倒か（車輪の再発明にならないか）
4. 基準を満たさない場合は、正直に「これはスキル向きではありません」と言って、理由を説明し、別の候補を探してください。無理にスキル化を勧めないでください。
5. 候補が1つに決まったら、最後に次の形式で「スキル企画メモ」を出力してください。

【スキル企画メモの形式】
・スキル名：（かんたんな日本語で）
・何をする作業か：（1〜2行）
・手順：（ざっくり3〜5ステップ）
・用意する見本（模範回答）：（何を1つ用意するか）
・期待する効果：（週に何分くらい浮きそうか）

このメモは、次回の講義に持っていって発表します。それでは、最初の質問からお願いします。
```

**できた！の条件**

- [ ] AIと相談した（1回でも複数回でもOK）
- [ ] 「スキル企画メモ」が1枚ある
- [ ] 次回、メモを見せながら1〜2分で話せる
